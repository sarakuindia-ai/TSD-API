Step 3 of 4
TSD Credentials (Optional)

If the partner has their own TSD CustomerNumber and Passcode, enter them here. Otherwise, the platform will use sandbox mock data until live credentials are provided.

ℹ Live vs. Sandbox Mode

When TSD credentials are provided, all API calls will be forwarded to weblink.tsdasp.net/requests/service.svc/ using the partner's CustomerNumber and Passcode. Without credentials, the platform returns realistic mock responses for development and testing.

⚠ Security Note

TSD credentials are encrypted at rest using AES-256 and never exposed via the API. All TSD requests are proxied through the gateway — partners only receive their RezCentral API key.
